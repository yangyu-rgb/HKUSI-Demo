# CrossBorder AI Demo Script

Target duration: 3–4 minutes.

## Opening — 20 seconds

“普通地图能规划交通，却看不到未来口岸排队的不确定性。CrossBorder AI 将口岸等待预测、众包校准和主动提醒放进同一个跨境决策流程。”

State clearly that the prototype uses deterministic local data and does not represent live border conditions.

## Scene 1: Border Pulse — 35 seconds

1. Open the homepage.
2. Point out the four port cards: 罗湖、福田、皇岗、深圳湾.
3. Compare current waits with the next two forecast points.
4. Mention the weather/transport buffer alert.

Key message: the product shows where congestion is heading, not only where it is now.

## Scene 2: AI Route Decision — 70 seconds

1. Use the preloaded trip: 香港大学 → 深圳南山科技园, arrival by 09:30.
2. Keep “稳妥均衡” and click “生成 AI 建议”.
3. Show the recommended port, end-to-end time, fee, confidence interval, and late-arrival risk.
4. Compare with one faster or cheaper alternative.
5. Explain that every result includes route assumptions and a visible demo disclaimer.

Key message: the system optimizes a decision under uncertainty instead of returning a single static route.

## Scene 3: Crowdsource Calibration — 45 seconds

1. Select 福田.
2. Enter a 12-minute actual wait and “畅通”.
3. Submit the report.
4. Show the +10 points message and the new item at the top of the live feed.
5. Run the route prediction again and note that crowdsource sample counts have increased.

Key message: user feedback closes the prediction loop and makes the system adaptive.

## Scene 4: Proactive and B2B Value — 45 seconds

1. Set a 09:30 smart reminder.
2. Show that the product can trigger departure, anomaly, and better-route alerts.
3. Generate the four-person enterprise dispatch example.
4. Point out employee count, average commute, high-risk count, and individual departure times.

Key message: the same prediction engine supports both individual commuters and organizations.

## Closing — 20 seconds

“CrossBorder AI 的商业重点不是向个人用户收导航费，而是把跨境等待风险转化为巴士、物流和企业 HR 可执行的调度服务。”
