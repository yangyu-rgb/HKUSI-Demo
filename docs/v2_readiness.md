# V2 模型开发前置检查

当前系统还没有训练或启用 V2。本阶段的目标是把每一条可用实际等待反馈追溯到产生它的统计预测和影子 AI 值，并防止合成数据或低质量反馈被误当作模型效果证据。

## 标签闭环

1. `POST /api/predict` 生成确定性的 `forecast_run_id`，并保存每个口岸的统计预测、影子输出、特征、数据版本和来源状态。
2. 路线页面将该运行 ID 与推荐口岸预填到众包页面。
3. `POST /api/crowdsource/report` 在验证口岸匹配后保存反馈；只有未过期、`high` 质量且该运行口岸尚无标签的记录会写入实际等待标签。
4. `python backend/scripts/export_training_snapshot.py` 在 `data/runtime/training_snapshots/` 生成 CSV 和 JSON 元数据。输出目录被 Git 忽略。

快照包含标签哈希、时间范围、真实日期时间切分、缺失字段风险、Provider 状态和每条标签的 `forecast_run_id` / 报告 ID。它可复核数据来源，但不能自动证明模型有效。

## 受控实验阈值

`GET /api/demo/v2-readiness` 和企业页使用相同检查：

- 至少 200 条高质量实际等待标签；
- 覆盖四个口岸；
- 至少 21 个独立日期和 8 个小时切片；
- 口岸、天气、日历和事件本地 Provider 均未降级；
- 训练、验证和测试必须按日期先后划分，禁止随机打散未来标签。

所有检查通过时，只表示可以开展本地受控实验，仍不表示可以替换统计预测器。生产晋级还必须具备经审核的真实数据源、独立回测、按口岸和时间切片的误差/校准比较、漂移监测和人工验收。
