# 真实数据源准入清单

本清单用于把候选真实数据源推进到可用于受控 V2 实验的状态。当前没有已批准的外部口岸、天气或交通数据源；任何候选源在完成以下审查前都不得写入训练快照，也不得配置为生产 Provider。

## 来源优先级

1. 有明确许可和稳定版本说明的官方数据。
2. 有书面使用范围、保留期限和质量责任的合作方数据。
3. 明确声明为实际现场、获得建模同意且通过质量规则的用户众包观察。

公开可访问不等于允许训练。网页抓取、转售数据或来源不明的汇总数据不能仅凭技术上可下载就进入项目。

## 准入门槛

每个候选数据源必须留下可审计记录，并由项目负责人确认：

- 数据所有者、获取方式、许可条款和允许的用途，包括是否允许模型训练、衍生特征和结果展示。
- 个人数据、位置数据和敏感字段的处理依据、建模同意、去标识化方式、保留期限和删除流程。
- 口岸、方向、通关类型、时区以及“等待开始/结束”的业务定义与项目一致。
- 采样频率、缺失率、重复率、异常值、历史回补、修订方式和数据版本可被测量。
- 可追溯到来源记录和采集时间；凭据不会写入仓库，失效时可安全降级到现有 Demo Provider。

只有同时通过授权、隐私、语义和质量审查的来源，才能标记为 `approved_for_controlled_experiment`。生产使用需要另行审批。

## 最小记录契约

真实等待记录至少包含：

- 稳定记录 ID、来源类型和来源版本；
- 口岸 ID、通关方向和通关类型；
- 等待开始、等待结束、观察/上报时间及带时区时间戳；
- 实际等待分钟、质量状态和去重依据；
- 对众包记录保存建模同意状态，对官方/合作方记录保存批准批次 ID；
- 若来自本系统预测闭环，保存 `forecast_run_id` 和报告 ID。

训练快照不导出直接用户标识。生成、演示和真实记录必须保留不同来源类型，禁止通过改名或合并文件绕过隔离。

## 接入步骤

1. 建立候选源审查记录并保存条款快照或合作方确认。
2. 用小规模只读样本完成字段映射、时区与等待定义核验。
3. 生成数据质量报告，人工抽查跨口岸、日期和小时切片。
4. 在隔离环境实现 Provider 或导入器，并测试缺失、重复、过期、修订和撤回。
5. 经负责人批准后写入带来源版本的训练候选区；首次仍只开展离线/影子实验。
6. 持续监测来源变化、分布漂移和授权状态，授权撤回时停止新增并按保留策略处理历史记录。

## 当前状态

- 外部官方/合作方来源：尚未批准或接入。
- 用户众包：已具备真实现场声明、方向/通道、等待起止时间和建模同意字段；仍属于自报观察。
- 生成历史：仅用于工程回归、压力情景和消融，不进入真实验证、测试或晋级指标。
